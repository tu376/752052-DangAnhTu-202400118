Q1: When should accessor method be used?

Answer:
- Use accessors only for data that truly needs to be read or written by other classes
- Prefer behavior methods over getters + setters
- When returning objects, return interfaces or immutable views
- In boundary / framework layers, accessors may be necessary
- Refactor out accessors when no longer needed



Q2: If you create a constructor method to build a DVD by title then create a constructor method to build a DVD by category. Does JAVA allow you to do this?

Answer:
- Java does NOT allow this. Because both constructors have the exact same signature, both parameters have String type
